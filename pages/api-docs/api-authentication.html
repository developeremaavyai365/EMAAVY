API Documentation

API Authentication

Authenticate every EMAAVY REST request with workspace-scoped API keys. Keys are issued per sub-account, rotate without downtime, and scope to granular permissions for agents, telephony, and knowledgebase operations.

01 Overview

Bearer tokens & workspace isolation

EMAAVY uses Bearer token authentication on all /v1 endpoints. Generate keys from the dashboard under Settings → API Keys or via the POST /v1/api-keys endpoint using a parent admin key.

Each key carries a sub_account_id claim. Requests made with a sub-account key can only access agents, numbers, campaigns, and knowledgebases owned by that tenant — enforcing multi-tenant isolation at the API gateway.

02 POST https://api.emaavy.com/v1/api-keys

Create an API key

Issue a scoped key for programmatic access.

Example request

curl -X POST https://api.emaavy.com/v1/api-keys \
  -H "Authorization: Bearer <EMAAVY_ADMIN_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "production-outbound",
    "sub_account_id": "sub_acct_8f2k9",
    "scopes": ["agents:read", "calls:write", "knowledgebases:read"],
    "expires_in_days": 90
  }'

Request body
