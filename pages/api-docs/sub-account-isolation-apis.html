API Documentation

Sub-account Isolation APIs

Provision isolated sub-accounts for agencies, portfolio companies, or business units — each with separate API keys, billing meters, and data residency controls.

01 Model

Hard tenant boundaries

Sub-accounts are first-class tenants. Agents, numbers, recordings, and vector indices are namespaced by sub_account_id. Cross-tenant reads return 404 — not 403 — to prevent enumeration.

Parent accounts can impersonate sub-accounts using X-EMAAVY-Act-As: sub_acct_8f2k9 for support workflows, audited in immutable logs.

02 POST https://api.emaavy.com/v1/sub-accounts

Create sub-account

Spin up a child tenant under your organization.

Example request

curl -X POST https://api.emaavy.com/v1/sub-accounts \
  -H "Authorization: Bearer <EMAAVY_PARENT_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Acme — APAC Sales",
    "billing_email": "finance@acme.com",
    "data_region": "ap-south-1"
  }'

Request body

{
  "name": "Acme — APAC Sales",
  "billing_email": "finance@acme.com",
  "data_region": "ap-south-1"
}

Responses

200 — OK — sub-account created
{
  "id": "sub_acct_8f2k9",
  "name": "Acme — APAC Sales",
  "data_region": "ap-south-1",
  "status": "active"
}
400 — Bad Request
{
  "error": {
    "code": "invalid_request",
    "message": "Missing required field or malformed JSON body.",
    "field": "agent_id"
  }
}
401 — Unauthorized
{
  "error": {
    "code": "unauthorized",
    "message": "Invalid or missing Authorization header. Use Bearer <EMAAVY_API_KEY>."
  }
}
501 — Not Implemented
{
  "error": {
    "code": "not_implemented",
    "message": "This endpoint is not enabled for your workspace tier. Contact sales@emaavy.com."
  }
